# 第二阶段：事件窗口与 SURGE 兼容派生数据

日期：2026-07-13

## 事件窗口分析

对20个事件的248,471条唯一帖子计算了完整覆盖、峰值日，以及连续21天、28天、31天的最高密度窗口。

- 全周期达到50,000条目标：1个事件；
- 28天窗口达到50,000条目标：1个事件；
- 达标事件：`FIFAWorldCup2026Kickoff`；
- 其28天高密度窗口包含87,922条帖子；
- 20事件28天候选窗口合计保留228,690条帖子。

自动窗口是数据密度候选，不替代人工事件语义确认。最终窗口应由项目负责人结合事件发生时间审核。

## 已生成的派生数据

对每个事件生成6H、12H、1D三种粒度，共60个事件—粒度目录：

```text
data/release/surge/events/<event>_<granularity>/
├── comment_count.csv
├── comment_count_normalized.csv
├── normalization.json
└── text_view.jsonl
```

并生成每事件关系目录：

```text
data/release/surge/events/<event>/
├── edges.jsonl
└── post_id_lookup.jsonl
```

## 处理规则

- 讨论量包括主帖、回复和引用帖，口径与“文本单元”一致；
- 时间箱以UTC对齐；
- 活跃窗口中的空箱保留为空值；
- z-score统计量只使用前70%训练时间段；
- 标准化CSV保持空箱为空值；
- 每箱最多选择3条主帖，每条最多选择最早2条同箱回复；
- 主帖按同箱回复量排序；
- 关系边仅保留两端帖子均在事件窗口内的记录；
- 发布关系文件不包含用户ID、昵称或位置字段。

## 情感状态

已经定义模型无关的情感标注契约和验证器，要求记录：

```text
post_id, event_id, platform, sentiment, sentiment_score,
model_name, model_version, prompt_version, processed_at, schema_version
```

标签限定为positive、neutral、negative，对应+1、0、-1。

当前没有生成`sentiment_polarity.csv`，因为尚未接入真实情感模型。项目不会使用规则或随机标签伪造缺失情感信号。
