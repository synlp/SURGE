# 项目范围与交付目标

## 背景

团队计划完成约 300 个事件的数据建设，其中已有约 100 个事件，后续主要处理约 200 个事件。事件生命周期通常控制在三周至一个月，允许少量特殊事件超过该范围。

每个事件目标约 50,000 个文本单元。文本单元包括主帖、回复、评论、引用等能够独立表达内容的一段文本，不等同于 50,000 条主帖。

## 核心交付

1. 多平台原始数据接入规范；
2. 统一帖子、互动边和情感记录范式；
3. 可复现的数据清洗、去重和事件裁剪管线；
4. 每个事件的质量、数量和来源覆盖报告；
5. SURGE 兼容派生数据导出器；
6. 可以独立测试、运行和交付的代码仓库。

## 每事件必须报告的数量

- 原始记录数；
- 主帖数；
- 回复/评论数；
- 引用/转发数；
- 非空文本单元数；
- 唯一帖子数；
- 去重前后数量；
- 各平台与各时间段覆盖；
- 解析失败、缺失字段和过滤数量。

## 设计原则

- 原始数据不可变；
- 所有派生记录可追溯到来源文件和来源记录；
- 平台差异通过适配器隔离；
- 中间范式不依赖 SURGE 的代码实现；
- SURGE 只是一个可选导出目标；
- 清洗规则、模型版本和配置全部可记录；
- 公开数据与受控原始数据严格分层。
